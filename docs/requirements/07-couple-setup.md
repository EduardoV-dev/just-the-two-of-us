# Requirement: Couple Setup

## Context

After a user signs in with Google for the first time (see `06-google-auth.md`), they have no partner linked. This onboarding flow lets them either create a new couple (generating an invite for their partner) or join an existing couple via an invite code. Once linked, the couple record stores the relationship start date used by the day counter (see `08-relationship-counter.md`).

## Goals

- Guide new users through linking with their partner immediately after first sign-in.
- Support two paths: **Create couple** (I'm the first one) and **Join couple** (my partner already set it up).
- Capture the relationship start date during couple creation.
- Block access to the rest of the app until the couple is set up.

## UI Spec

### Couple Setup Page (`/couple/setup`)

Shown automatically when `AuthUser.coupleId === null`. Not accessible once a couple is linked (redirect to `/`).

**Step 1 — Choose path**

Two large cards side by side (or stacked on mobile):

| Card | Label | Description |
|------|-------|-------------|
| A | Create a couple | I'll invite my partner |
| B | Join a couple | I have an invite code |

**Step 2A — Create couple**

- Date picker labeled "When did your relationship start?" (required, no future dates).
- "Create & get invite code" button.
- On success: display the generated invite code in a styled box with a **Copy** button and a short explanation ("Share this code with your partner so they can join").
- A "Continue to app" button appears once the code is shown (partner can join later).

**Step 2B — Join couple**

- Single text input labeled "Invite code" with a placeholder like `XXXX-XXXX`.
- "Join" button.
- On success: redirect to `/`.
- On invalid code: inline error "Code not found or already used."

### Loading & Error States

- Button shows spinner while the API call is in-flight.
- Generic network error shown inline with a "Try again" option.

## Data Shape (Frontend)

```ts
interface Couple {
  id: string;
  partnerAId: string;
  partnerBId: string | null;   // null until second partner joins
  startDate: string;           // ISO 8601 date, e.g. "2022-03-14"
  inviteCode: string;          // [PLACEHOLDER — generated by backend]
}
```
